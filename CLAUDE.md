# Project Context: Braxton Hawkins Personal Site

This is a premium personal brand website built with Next.js, TypeScript, Tailwind, shadcn/ui, and Framer Motion.

It represents a dual identity:
- Software Engineer (systems, structure, logic)
- DJ / Curator (groove, energy, atmosphere)

The site should feel like a:
digital archive of systems and sound.

---

## Design Principles

- Editorial, not SaaS
- Minimal, not empty
- Expressive, not loud
- Structured, not boxed
- Premium, not trendy

Avoid:
- generic portfolio layouts
- excessive cards
- glassmorphism
- bright neon cyberpunk
- overly rounded UI

---

## Design System

- 8px spacing system
- 12-column grid
- asymmetrical layouts (7/5, 8/4)
- thin borders (1px)
- minimal radius (0–8px)
- no heavy shadows
- subtle grain allowed

---

## Component Philosophy

- Avoid “card soup”
- Use layout, spacing, and typography for structure
- Cards only when necessary (projects, music, posts)

---

## Visual Modes

### Dark Mode (primary)
- cinematic, archival
- used for hero, projects, music

### Light Mode (secondary)
- editorial, gallery-like
- used for about, writing, contact

---

## Typography

- Display: editorial serif
- UI: modern sans
- Metadata: monospace

---

## Signature Elements

- section indexing (01, 02, 03)
- system labels (e.g. [ ENG // DJ ])
- archive / volume language

---

## Motion

- subtle only
- 0.4–0.8s
- opacity + translate
- no bounce or flashy animations

---

## Taste Standard

Every change should feel:
- intentional
- restrained
- cohesive
- like a designed system

If something feels generic, simplify and refine.
